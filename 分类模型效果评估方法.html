机器学习中怎样评估分类模型的效果？

在现实生活中，许多事件的发生都是具有随机性的，比如我们无法准确确定股价涨跌、天气变化情况等，而只能通过概率来推测事件发生的可能性。

在机器学习中，逻辑回归是一种计算概率的有效机制，逻辑回归模型可以预测概率作为输出结果，帮助我们对样本所属标签进行判定。

在二元分类任务中，通常将逻辑回归值映射到二元类别，转换为二进制值之后进行类别划分。可以使用sigmoid函数对输出的y进行线性变换，在y趋向正无穷大时，转换后的输出概率会趋近于1，在y 趋向负无穷大时，转换后的输出概率会趋近于0，从而可以实现分类预测。

现在比较关心的问题是，如何将逻辑回归值映射到二元类别呢？

如果要对事件进行分类，就需要借助相关的判定标准，分类模型也是如此。

我们必须定义一个分类阈值，高于该阈值的值表示为目标值所属类，而低于该阈值的值则表示为另一类。

至于模型分类的效果如何，就需要借助一些专门的评价指标了。

评价指标

可用于评估分类模型预测效果的指标有准确率（accuracy）、精确率（precision）、召回率（recall）和F1值。

先来看这样一个图表，它被称之为混淆矩阵，每一列代表预测值，每一行代表实际类别，在这个表格中我们可以清楚看到每个类别被正确预测和错误识别的数量。

因为混淆矩阵可以清晰地呈现各类别之间是否存在混淆，因此衡量的是一个分类器分类的准确程度。

在具体介绍准确率和召回率之前，先来认识一下什么是真阳性、假阳性、真阴性和假阴性。

真阳性（TP）是指实际为正类且模型预测为正类，假阳性（FP）是指实际为负类而模型将其预测为正类，真阴性（TN）是指实际为负类而模型预测为负类，假阴性（FN）是指实际为正类而模型将其预测为负类。

准确率（accuracy）等于模型正确预测的数量与全部数量之比：

accuracy = （TP+TN）/(P+N)

在实际分类中，准确率越高就表示模型性能越好吗？未必如此。


下面举个例子，比如有一个可以识别垃圾邮件的过滤器模型，它会将邮件分类为垃圾邮件（阳性类）或非垃圾邮件（阴性类）：

真阳性（TP）：
实际：垃圾邮件预测：垃圾邮件TP数量：2 		假阳性（FP）：
实际：非垃圾邮件预测：垃圾邮件FP数量：2
假阴性（FN）：
实际：垃圾邮件预测：非垃圾邮件FN数量：6 		真阴性（TN）：
实际：非垃圾邮件预测：非垃圾邮件TN数量：90

Accuracy = （TP+TN）/(TP+TN+FP+FN) = （2 + 90）/（2+90+2+6）= 0.92

准确率达到0.92，即100个样本中有92个的类别预测正确。

这个结果看似很好，但再进一步分析，100个样本中，实际上的垃圾邮件有8个，而模型预测正确的只有2个，还有6个没有识别出来。

试想一下，如果邮件数量巨多，那使用这样模型的过滤器又怎么能胜任垃圾邮件拦截工作呢？

因此，仅仅参考准确率（accuracy）是不够的，还需要同时借助精确率（precision）和召回率（recall）等其他评价指标。

精确率（precision）等于模型正确预测为阳性的数量与模型预测为阳性的总数之比：

Precision = TP /（TP+FP）

在上面的例子中，Precision = TP /（TP+FP）= 2 /（2+2）= 0.5，即模型预测为垃圾邮件时，有50%的概率是正确的。

召回率（recall）等于模型正确预测为阳性的数量与实际为阳性的总数之比：

recall = TP /（TP + FN）

所以示例中模型的召回率 recall = TP /（TP + FN）= 2 /（2+6）= 0.25，即模型正确识别了所有垃圾邮件的25%。


现在有一个问题，精确率（precision）和召回率（recall），两者有什么关系？哪个更重要一些呢？

实际上，精确率和召回率是成反比的，即精确率越高，召回率越低，而且还会受到分类阈值的影响。


下面我们对示例中的样本数量作调整，重新计算精确率和召回率：

真阳性（TP）：20 假阳性（FP）：3
假阴性（FN）：5 真阴性（TN）：72

Precision = TP /（TP+FP）= 20 /（20+3）= 0.86

Recall = TP /（TP + FN）= 20 /（20+5）= 0.8

再尝试提高分类阈值，增加假阴性数量，减少假阳性数量：

真阳性（TP）：18 假阳性（FP）：1
假阴性（FN）：7 真阴性（TN）：74

Precision = TP /（TP+FP）= 18 /（18+1）= 0.94

Recall = TP /（TP + FN）= 18 /（18+7）= 0.72

可以发现，精确率从0.86提高到0.94的同时，召回率却从0.8降低到了0.72。


再来降低分类阈值，减少假阴性数量，增加假阳性数量：

真阳性（TP）：22 假阳性（FP）：5
假阴性（FN）：3 真阴性（TN）：70

Precision = TP /（TP+FP）= 22 /（22+5）= 0.81

Recall = TP /（TP + FN）= 22 /（22+3）= 0.88

结果精确率从0.94又下降到了0.81，而召回率却从0.72提高到了0.88。

ROC曲线和AUC

最后，提到分类阈值，就不得不说和它密切相关的ROC曲线。

ROC曲线（receiver operating characteristic curve）的全称是接收者工作特征曲线，用来表示分类模型中不同分类阈值的性能。它的横轴为假阳率（FPR），FPR=FP/(FP+TN)，即实际的负类样本中模型错误预测为正类的比例；纵轴为真阳率（TPR），TPR=TP/(TP+FN)，即实际的正类样本中模型正确预测的比例。



AUC是指ROC曲线下方的面积，面积为0.5时为随机分类，表示模型没有识别能力，面积越接近于1表明模型的识别能力越强，面积等于1为完全识别。

现实世界中二元分类的 AUC 值通常是在0.5到1之间，但实际上，如果AUC的值为1，并不意味着模型分类效果达到最好。相反，可能说明模型存在过拟合或者标签数据和特征数据有重合等的错误。
